# DESIGN.md

## 产品定位

AI 产品交付工作台 - 面向产品经理的交付效率工具，不是 AI 创作工具。

## 设计原则

1. **清晰胜于美观** - 功能结构图、线框图的目标是清晰表达，不是视觉设计
2. **保守标注不确定** - 对不确定的内容明确标注"推测"、"待确认"
3. **产品经理友好** - 符合产品经理的工作习惯和思维模式
4. **研发可接手** - 输出足够详细，研发无需反复沟通

## 配色方案

采用中性灰调为主，突出信息层级：

- **主色**: 灰蓝系 (#3B82F6 blue-500)
- **成功/确认**: 绿色 (#22C55E green-500)
- **警告/待确认**: 橙色 (#F59E0B amber-500)
- **错误/不确定**: 红色 (#EF4444 red-500)
- **背景**: 浅灰 (#F9FAFB gray-50)
- **边框**: 中灰 (#E5E7EB gray-200)

## 功能节点类型颜色

| 类型 | 颜色 | 说明 |
|------|------|------|
| page | blue | 页面 |
| module | violet | 模块 |
| field | slate | 字段 |
| action | green | 操作 |
| modal | amber | 弹窗 |
| bottom-sheet | cyan | 底部弹层 |
| state | orange | 状态 |
| rule | purple | 规则 |

## 置信度标识

| 置信度 | 颜色 | 说明 |
|--------|------|------|
| high | green | 已识别，确定 |
| medium | amber | 推测，需确认 |
| low | red | 不确定，必须确认 |

## 状态标识

| 状态 | 颜色 | 说明 |
|------|------|------|
| confirmed | green | 已确认 |
| pending | amber | 待确认 |
| optional | slate | 可选 |
| later | gray | 后续版本 |

## 布局结构

### 功能结构图确认页（核心页面）

采用三栏布局：

```
┌─────────────────────────────────────────────────────────────┐
│                        顶部导航栏                            │
├────────────────┬────────────────────────┬──────────────────┤
│                │                        │                  │
│   功能结构树    │      节点详情          │    待确认问题     │
│   (可展开/折叠) │      (编辑面板)        │    (问答面板)     │
│                │                        │                  │
│   左侧 1/4     │      中间 2/4          │    右侧 1/4      │
│                │                        │                  │
└────────────────┴────────────────────────┴──────────────────┘
```
